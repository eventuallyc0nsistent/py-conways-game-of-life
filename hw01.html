CS9053 Introduction to Java

Homework 1

  • Install the Java jdk on your computer.
  • Edit, compile and run a Hello World style program on your own computer. (Don't hand this in.)
  • Read The Java Language: An Overview
  • Read at least the first two chapters in Core Java.
  • Implement Conway's game of life in a language other than Java. (You can find lots of other info on the web.)
    • Do not "query the user" for anything. The grader does not want to have to type in anything after he starts running your program.
    • Get the initial state from a file. Assume the file's name is "life.txt" and is in the current working directory.
      No, do not hardcode an absolute pathname!
      • The first line of the file will have two integers,
        • the first is how many rows there are in the world
        • the second is how many columns there are in the world
      • Each row will have spaces and asterisks.
        • Spaces represent empty cells
        • Asterisks represent occupied cells
      • If a row is "short", then assume the missing characters are blank.
      • If a row is "long", then ignore the extra characters.
      • If there are not enough rows in the file, then assume that the missing rows should be filled with empty cells.
    • Display each "generation" of the game to standard output (ie. the screen). Generate 10 generations beyond the one you read from the file.
    • Feel free to use command line parameters, if you like, to allow other input files or generation counts to be used. This is not required, nor is it extra credit, but you may find it convenient while testing.
    • The rules of life say:
      • A living cell with two or three neighboring living cells survives into the next generation. A living cell with fewer than two or more than three living neighbors will die.
      • Each empty/dead cell that has exactly three living neighbors--no more, no fewer--is a birth cell. It will be alive in the next generation.
    • Suggestion: Create an extra row on the top and bottom as well as an extra column to the right and the left of the actual data. Mark the extra cells as dead. This will simplify your code, as you won't need special checks for "boundary" cells.
